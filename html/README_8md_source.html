3 ## Installation
4 In this directory the [CMakeLists.txt](ubcc1pi_standalone/CMakeLists.txt) file is used when compiling the package using LArSoft. To run the
5 analysis code outside of LArSoft, just start up root and compile the required code using the instruction below.
6 
7 Run the set-up script for the standalone environment
8 ```
9 source setup_standalone.sh
10 ```
11 
12 This script sets the environment variable `UBCC1PI_STANDALONE_DIR` and makes an alias (`ubcc1pi = "root -l -b compile.C"`) that starts up
13 root and compiles the required standalone code. If the code is already compiled then `ubcc1pi` will just start root and load it.
14 ```
15 ubcc1pi
16 ```

33 ## Modifying the configuration
34 Start root and load the compiled code:
35 ```
36 ubcc1pi
37 ```
38 
39 The configuration options are handled by the `ubcc1pi::Config` structure. Make an instance of it using the root command line:
40 ```
41 ubcc1pi::Config config;
42 ```
43 
44 Print the configuration to see the defaults
45 ```
46 ubcc1pi_macros::PrintConfig(config);
47 ```
48 
49 You can modify a parameter as follows - for a full list of parameters, see
50 [ubcc1pi_standalone/Objects/Config.h](ubcc1pi_standalone/Objects/Config.h). You can also press TAB to see the options.
51 ```
52 config.files.overlaysFileName = "/path/to/my/file.root";
53 ```
54 
55 To run a macro with your custom configuration just pass it as the first argument. Please note that macros can create files and plots in the
56 current directory so you should navigate to a suitable directory before running a macro.
57 ```
58 ubcc1pi_macros::PrintConfig(config);
59 ```
60 
61 If you want to change the default configuration, then just modify [ubcc1pi_standalone/Objects/Config.h](ubcc1pi_standalone/Objects/Config.h).

64 ## Writing a new macro
65 First declare the macro in [ubcc1pi_standalone/Macros/Macros.h](ubcc1pi_standalone/Macros/Macros.h). For example:
66 ```
67  /**
68  * @brief Description of my new macro
69  *
70  * @param config the input config
71  */
72  void MyNewMacro(const Config &config = Config());
73 ```
74 
75 By convention, the macro should only take one argument which is the `ubcc1pi::Config` object. Use `const Config &config = Config()` so the
76 default configuration can be easily used. You can add any configuration options required into
77 [ubcc1pi_standalone/Objects/Config.h](ubcc1pi_standalone/Objects/Config.h). If you modified the configuration, then make sure that that the
78 [ubcc1pi_standalone/Macros/PrintConfig.cxx](ubcc1pi_standalone/Macros/PrintConfig.cxx) prints includes your new variables too.
79 
80 Now add the actual macro in the [ubcc1pi_standalone/Macros/](ubcc1pi_standalone/Macros/) directory, take a look at the existing macros as an
81 example - you should use the `ubcc1pi_macros` namespace.
82 
83 Finally, add the macro to the compilation script [ubcc1pi_standalone/compile.C](ubcc1pi_standalone/compile.C) - you just need to add the
84 path to your macro with the rest. Now your macro will be compiled along with the rest when you run `ubcc1pi`.
